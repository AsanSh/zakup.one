# Техническое задание (ТЗ) на разработку Административной части платформы ZAKUP.ONE

## 1. Общие сведения

**Проект:** Административная панель платформы для снабженцев строительных компаний  
**Цель:** Централизованное управление товарами, поставщиками, клиентами и заявками  
**Роль:** Администратор системы (наша команда)  
**Технологии:** React 18, TypeScript, Vite, Tailwind CSS, FastAPI (Python), PostgreSQL  
**Репозиторий:** https://github.com/AsanSh/zakup.one.git

## 2. Архитектура проекта

### 2.1. Разделение на модули

Проект четко разделен на три части:

```
frontend/src/
├── admin/          # Административная часть (МЫ - видим все, управляем всем)
│   ├── components/ # Компоненты админ-панели
│   ├── pages/      # Страницы админ-панели
│   └── hooks/      # Хуки для админки
├── client/         # Клиентская часть (снабженцы - только свои данные)
│   ├── components/ # Компоненты клиентской части
│   ├── pages/      # Страницы для клиентов
│   └── hooks/      # Хуки для клиентов
└── shared/         # Общие компоненты и утилиты
    ├── components/ # Общие компоненты (Login, Register)
    ├── api/        # API клиенты
    ├── types/      # TypeScript типы
    └── utils/      # Утилиты
```

### 2.2. Роутинг

- **Админ-панель:** `/admin/*` - доступна только администраторам
- **Клиентская часть:** `/search`, `/cart`, `/orders`, `/profile` - доступна верифицированным клиентам
- **Публичные:** `/login`, `/register` - доступны всем

## 3. Функциональные требования для админ-панели

### 3.1. Панель управления (Dashboard) - `/admin`

**Цель:** Обзор системы в реальном времени

**Компоненты:**
- **Статистические карточки:**
  - Новые заявки (количество)
  - Клиенты на модерации (количество)
  - Активные поставщики (количество)
  - Общий оборот за месяц (сумма)
  
- **График активности заявок:**
  - График заявок по дням за последние 30 дней
  - Использовать Chart.js или Recharts
  
- **Последние заявки:**
  - Таблица последних 10 заявок
  - Колонки: №, Клиент, Сумма, Дата, Статус
  - Клик по строке → детальный просмотр
  
- **Быстрые действия:**
  - Кнопка "Загрузить прайс-лист"
  - Кнопка "Модерация клиентов"
  - Кнопка "Обновить цены"

**API Endpoint:** `GET /api/v1/admin/stats`

### 3.2. Управление товарами - `/admin/products`

**Цель:** Полный контроль над каталогом товаров

**Функционал:**

1. **Таблица товаров:**
   - Колонки: Чекбокс, Наименование, Категория, Ед.изм., Цена, Остаток, Поставщик (скрыт от клиентов), Статус
   - Пагинация (50/100/200 строк на страницу)
   - Сортировка по колонкам
   - Множественный выбор товаров
   
2. **Фильтры:**
   - Поиск по наименованию/артикулу
   - Фильтр по категории
   - Фильтр по поставщику
   - Фильтр по статусу (активен/неактивен)
   
3. **Массовые операции:**
   - Массовое изменение цен (процент или фиксированная сумма)
   - Массовое скрытие/показ товаров
   - Массовое удаление
   - Экспорт в Excel/CSV
   
4. **Редактирование товара:**
   - Модальное окно или отдельная страница
   - Поля: Наименование, Артикул, Категория, Ед.изм., Цена, Страна производства, Поставщик, Статус
   - Валидация всех полей
   
5. **Импорт прайс-листов:**
   - Загрузка файла (XLSX, CSV)
   - Сопоставление колонок
   - Предпросмотр данных
   - Настройки импорта (обновлять существующие, деактивировать отсутствующие)
   - История импортов

**API Endpoints:**
- `GET /api/v1/admin/products` - список товаров
- `PUT /api/v1/admin/products/{id}` - обновление товара
- `POST /api/v1/admin/products/bulk-update` - массовое обновление
- `POST /api/v1/admin/products/import` - импорт из файла

### 3.3. Управление поставщиками - `/admin/price-lists/management/suppliers`

**Цель:** Управление поставщиками и их прайс-листами

**Функционал:**

1. **Список поставщиков:**
   - Таблица: ID, Название, Email, Телефон, Количество товаров, Последнее обновление, Статус
   - Фильтры: Поиск, Статус (активен/неактивен)
   
2. **Добавление/редактирование поставщика:**
   - Форма: Название, Email, Телефон, Статус
   - Валидация email и телефона
   
3. **Управление прайс-листами поставщика:**
   - Список настроенных автоматических обновлений
   - Настройка URL для скачивания
   - Настройка частоты обновления (вручную, ежедневно, еженедельно, ежемесячно)
   - Параметры парсинга (номер строки заголовка, номер строки начала данных)
   - История обновлений
   - Ручной запуск обновления

**API Endpoints:**
- `GET /api/v1/admin/suppliers` - список поставщиков
- `POST /api/v1/admin/suppliers` - создание поставщика
- `PUT /api/v1/admin/suppliers/{id}` - обновление поставщика
- `POST /api/v1/admin/price-lists/download-and-import` - скачать и импортировать прайс-лист
- `GET /api/v1/admin/price-lists/updates` - список автоматических обновлений
- `PUT /api/v1/admin/price-lists/updates/{id}` - обновление настроек
- `POST /api/v1/admin/price-lists/updates/{id}/run` - запуск обновления

### 3.4. Управление клиентами (снабженцами) - `/admin/users`

**Цель:** Модерация и управление клиентами

**Функционал:**

1. **Вкладка "На одобрении":**
   - Список заявок на регистрацию
   - Информация: Email, ФИО, Компания, Телефон, Дата регистрации
   - Действия: Одобрить, Отклонить (с указанием причины)
   - При одобрении → отправка email-уведомления
   - При отклонении → отправка email с причиной
   
2. **Вкладка "Все клиенты":**
   - Список всех верифицированных клиентов
   - Информация: ID, Email, ФИО, Компания, Телефон, Статус (верифицирован/активен), Дата регистрации, Последний заказ
   - Действия: Активировать/Деактивировать, Просмотр финансов, История заказов
   
3. **Финансовый раздел клиента:**
   - Лимит на рассрочку
   - Информация о залоге (наименование, документы, оценочная стоимость)
   - Статистика: Всего товаров, Покупки наличными/в кредит, Остаток кредита, Сумма договора, Платежи, Последняя транзакция, Статус

**API Endpoints:**
- `GET /api/v1/admin/users/pending` - список на модерации
- `POST /api/v1/admin/users/{id}/verify` - одобрить клиента
- `POST /api/v1/admin/users/{id}/reject` - отклонить клиента (с причиной)
- `GET /api/v1/admin/users` - список всех клиентов
- `POST /api/v1/admin/users/{id}/activate` - активировать
- `POST /api/v1/admin/users/{id}/deactivate` - деактивировать
- `GET /api/v1/admin/users/{id}/finance` - финансовая информация

### 3.5. Управление заявками - `/admin/orders`

**Цель:** Обработка и отслеживание заявок клиентов

**Функционал:**

1. **Список заявок:**
   - Таблица: №, Клиент, Объект доставки, Сумма, Дата создания, Статус, Действия
   - Фильтры: По статусу, По дате, По клиенту, По сумме
   - Сортировка: По дате, По сумме
   - Быстрый поиск по номеру заявки или клиенту
   
2. **Детальный просмотр заявки:**
   - Модальное окно или отдельная страница
   - Информация о клиенте: ФИО, Компания, Контакты
   - Информация об объекте: Адрес, Комментарий, Желаемая дата доставки, Контактное лицо
   - Список товаров: Наименование, Количество, Ед.изм., Цена, Сумма
   - Итого по заявке
   - История изменения статусов
   - Комментарии к заявке
   
3. **Изменение статуса:**
   - Выбор статуса: Новая → В обработке → Собрана → Отправлена → В пути → Доставлена
   - Возможность отмены заявки
   - Указание номера отслеживания
   - Указание ожидаемой даты доставки
   - Комментарий к изменению статуса
   
4. **Отслеживание доставки:**
   - Просмотр статуса доставки
   - История событий доставки
   - Интеграция с трекингом

**API Endpoints:**
- `GET /api/v1/admin/orders` - список заявок
- `GET /api/v1/admin/orders/{id}` - детали заявки
- `PUT /api/v1/admin/orders/{id}/status` - изменение статуса
- `GET /api/v1/admin/orders/stats` - статистика по заявкам

### 3.6. Прайс-листы - `/admin/price-lists`

**Цель:** Управление загрузкой и обновлением прайс-листов

**Функционал:**

1. **Загрузка прайс-листа:**
   - Выбор поставщика
   - Загрузка файла (XLSX, CSV) или указание URL
   - Настройка параметров парсинга
   - Настройка частоты автоматического обновления
   - Предпросмотр данных перед импортом
   
2. **Список автоматических обновлений:**
   - Таблица: Поставщик, URL, Частота, Последнее обновление, Следующее обновление, Статус, Результаты
   - Действия: Изменить настройки, Запустить сейчас, Активировать/Деактивировать
   
3. **История импортов:**
   - Список всех импортов
   - Результаты: Сколько добавлено, Сколько обновлено, Ошибки
   - Просмотр логов ошибок

**API Endpoints:**
- `POST /api/v1/admin/price-lists/download-and-import` - скачать и импортировать
- `GET /api/v1/admin/price-lists/updates` - список обновлений
- `PUT /api/v1/admin/price-lists/updates/{id}` - обновить настройки
- `POST /api/v1/admin/price-lists/updates/{id}/run` - запустить обновление
