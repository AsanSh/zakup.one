# Исправления проблем производительности

## Проблемы, которые были исправлены:

### 1. Бесконечная инициализация authStore
**Проблема:** `init()` вызывался многократно, создавая лишние запросы
**Решение:** 
- Добавлен флаг `isInitialized` в состояние
- Проверка перед вызовом `init()` в `main.tsx`
- Проверка внутри `init()` для предотвращения повторных вызовов

### 2. Использование axios вместо apiClient
**Проблема:** В `authStore` использовался глобальный `axios`, а не настроенный `apiClient`
**Решение:** Заменен `axios` на `apiClient` во всех местах

### 3. Отсутствие loading state при инициализации
**Проблема:** Защищенные роуты рендерились до завершения инициализации
**Решение:** 
- Добавлена проверка `isInitialized` в `ProtectedRoute` и `AdminRoute`
- Показывается loading spinner во время инициализации

### 4. Потенциальные бесконечные редиректы
**Проблема:** `Navigate` без `replace` мог создавать циклы
**Решение:** Добавлен `replace` ко всем `Navigate` компонентам

## Файлы, которые были изменены:

1. `frontend/src/store/authStore.ts`
   - Добавлен `isInitialized: boolean`
   - Заменен `axios` на `apiClient`
   - Добавлена проверка в `init()`

2. `frontend/src/App.tsx`
   - Добавлена проверка `isInitialized` в `ProtectedRoute` и `AdminRoute`
   - Добавлен loading spinner
   - Добавлен `replace` к `Navigate`

3. `frontend/src/main.tsx`
   - Добавлена проверка перед вызовом `init()`
