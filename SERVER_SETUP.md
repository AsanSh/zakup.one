# Настройка сервера после загрузки файлов

## ✅ Файлы загружены на сервер

Все файлы успешно загружены в папку `zakup.one` на сервере Spaceship.

## 📋 Следующие шаги в панели управления Spaceship:

### 1. Создайте .env файл

На сервере в папке `zakup.one/`:
- Скопируйте `.env.example` в `.env`
- Отредактируйте `.env` и измените:
  - `SECRET_KEY` - на случайную строку (минимум 32 символа)
  - `CORS_ORIGINS` - убедитесь что содержит `https://www.zakup.one` и `https://zakup.one`
  - `DATABASE_URL` - если используете PostgreSQL, настройте подключение

### 2. Настройте Python приложение

В панели управления Spaceship:
1. Найдите настройки Python приложения
2. Укажите:
   - **Entry Point (точка входа):** `wsgi:application`
   - **Рабочая директория:** `zakup.one` или полный путь к папке
   - **Версия Python:** 3.11 или доступная на сервере

### 3. Установите зависимости

Выполните в терминале сервера или через панель управления:
```bash
cd zakup.one
pip install -r requirements.txt
```

### 4. Настройте права доступа

Установите права на папки:
```bash
chmod 755 zakup.one
chmod 755 zakup.one/app
chmod 755 zakup.one/frontend
chmod 777 zakup.one/uploads  # Для загрузки файлов
chmod 644 zakup.one/*.py
chmod 644 zakup.one/.htaccess
```

### 5. Создайте базу данных (если нужно)

Если используете SQLite:
```bash
# База данных будет создана автоматически при первом запуске
# Убедитесь что папка доступна для записи
```

Если используете PostgreSQL:
- Создайте базу данных в панели Spaceship
- Настройте `DATABASE_URL` в `.env`

### 6. Настройте домен

В панели Spaceship:
1. Добавьте домен `www.zakup.one` или `zakup.one`
2. Настройте DNS записи (если нужно):
   - A запись: IP адрес сервера Spaceship
   - CNAME для www: основной домен

### 7. Настройте SSL сертификат

После настройки SSL:
1. Раскомментируйте правила HTTPS в `.htaccess`:
   ```
   RewriteCond %{HTTPS} off
   RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]
   ```

### 8. Перезапустите приложение

В панели Spaceship перезапустите Python приложение.

## 🔍 Проверка работы

После настройки проверьте:

1. **Главная страница:**
   - https://www.zakup.one
   - https://zakup.one

2. **API Health Check:**
   - https://www.zakup.one/api/v1/health
   - Должен вернуть: `{"status": "ok"}`

3. **API Документация:**
   - https://www.zakup.one/api/docs

4. **Frontend:**
   - Должна загружаться главная страница приложения

## ⚠️ Важные замечания

- **SECRET_KEY**: ОБЯЗАТЕЛЬНО измените на случайную строку в `.env`!
- **База данных**: Убедитесь что файл `zakup.db` (если используется SQLite) доступен для записи
- **CORS**: Проверьте что `CORS_ORIGINS` в `.env` содержит ваш домен
- **Логи**: Проверяйте логи в панели Spaceship при возникновении проблем

## 🐛 Решение проблем

### Ошибка 500
- Проверьте логи в панели Spaceship
- Убедитесь что все зависимости установлены: `pip install -r requirements.txt`
- Проверьте права доступа к файлам
- Проверьте `.env` файл

### API не работает
- Проверьте `.htaccess` файл
- Убедитесь что `wsgi.py` настроен правильно
- Проверьте что Python приложение запущено

### Статические файлы не загружаются
- Проверьте путь к `frontend/dist/`
- Убедитесь что файлы загружены
- Проверьте права доступа

### База данных не работает
- Проверьте `DATABASE_URL` в `.env`
- Убедитесь что база данных создана
- Проверьте права доступа к файлу базы данных

## 📞 Поддержка

Если возникли проблемы:
1. Проверьте логи в панели Spaceship
2. Убедитесь что все шаги выполнены
3. Проверьте документацию Spaceship
