# Быстрое тестирование авторизации

## 🚀 Быстрый старт

### Шаг 1: Включите упрощенную версию

В файле `frontend/src/main.tsx` замените:
```typescript
import App from './App.tsx'
```

На:
```typescript
import App from './App.simple.tsx'
```

### Шаг 2: Запустите приложение

```bash
cd frontend
npm run dev
```

### Шаг 3: Откройте браузер

1. Откройте http://localhost:5467/login
2. Откройте DevTools (F12) → Console
3. Попробуйте войти с:
   - Email: `admin@zakup.one`
   - Пароль: `admin123`

Или нажмите кнопку **"🧪 Тест: Войти как админ (без API)"**

## 🔍 Что проверять

### 1. Консоль браузера (F12 → Console)

Должны появиться логи:
- `🔍 Trying to login with: { email, password }`
- `✅ Login successful, user: {...}`
- `🔄 Redirecting to /admin`

### 2. LocalStorage (F12 → Application → Local Storage)

После входа должен появиться ключ `auth-storage` с данными пользователя.

### 3. URL

После успешного входа URL должен измениться на `/admin` (для админа) или `/search` (для пользователя).

## 🧪 Тестовая кнопка

Кнопка **"🧪 Тест: Войти как админ (без API)"** обходит API и:
- Создает тестового пользователя
- Сохраняет в localStorage
- Редиректит на `/admin`

Используйте её если API недоступен.

## 🔧 Отладка

### Проверить данные в консоли

Нажмите кнопку **"🔍 Проверить данные в консоли"** - выведет:
- LocalStorage
- Auth Store
- Текущего пользователя

### Очистить данные

Нажмите кнопку **"🗑️ Очистить LocalStorage"** для полной очистки.

## ✅ После успешного тестирования

Верните оригинальный импорт в `main.tsx`:
```typescript
import App from './App.tsx'
```

## 📝 Примечания

- Упрощенная версия использует те же компоненты (ClientLayout, AdminLayout)
- Все логирование выводится в консоль браузера
- Тестовая кнопка работает без backend
