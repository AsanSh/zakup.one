# Решение ошибки 500

## 🔍 Возможные причины ошибки 500:

1. **Зависимости не установлены** - нужно выполнить "Run Pip Install"
2. **Переменные окружения не настроены** - нужно добавить через "+ ADD VARIABLE"
3. **База данных не создана** - SQLite файл может отсутствовать
4. **Неправильные пути к файлам** - пути могут отличаться на сервере
5. **Ошибки импорта модулей** - некоторые модули могут отсутствовать

## ✅ Что было исправлено:

1. **Улучшен wsgi.py:**
   - Добавлена обработка ошибок при импорте
   - Правильная настройка путей
   - Автоматическая загрузка .env файла

2. **Улучшен health endpoint:**
   - Добавлен простой `/health` без подключения к БД
   - Добавлен `/api/v1/health` с проверкой БД

## 📋 Проверка в панели Spaceship:

### 1. Проверьте логи приложения

В панели Spaceship найдите раздел "Logs" или "Application Logs" и посмотрите ошибки.

### 2. Убедитесь что зависимости установлены

В секции "Configuration files":
- Должно быть: `requirements.txt`
- Нажмите "Run Pip Install" если еще не нажимали

### 3. Проверьте переменные окружения

Убедитесь что добавлены:
- `DATABASE_URL` = `sqlite:///./zakup.db`
- `SECRET_KEY` = (значение из .env)
- `DEBUG` = `False`

### 4. Создайте базу данных (если нужно)

Если используете SQLite, база данных создастся автоматически при первом запросе.
Убедитесь что папка доступна для записи.

### 5. Проверьте права доступа

Через SSH или панель управления:
```bash
cd zakup.one
chmod 755 .
chmod 755 app
chmod 777 uploads
```

## 🐛 Если ошибка сохраняется:

1. Проверьте логи в панели Spaceship
2. Убедитесь что все зависимости установлены
3. Проверьте что переменные окружения добавлены
4. Убедитесь что Python версия правильная (3.11)
5. Проверьте что Entry Point указан правильно: `wsgi:application`
