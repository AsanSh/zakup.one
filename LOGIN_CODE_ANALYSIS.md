## 🔍 КАК РАБОТАЕТ ЛОГИН:

### Flow:

1. **Login.tsx** → вызывает `login(email, password)` из `authStore`
2. **authStore.login()** → вызывает `authApi.login(email, password)`
3. **authApi.login()** → отправляет POST запрос на `/api/v1/auth/login`
4. **Backend** → обрабатывает запрос и возвращает токен
5. **authApi.login()** → проверяет ответ и возвращает данные
6. **authStore.login()** → сохраняет токен и пользователя
7. **Login.tsx** → редиректит на нужную страницу

### Формат запроса:

```typescript
POST /api/v1/auth/login
Content-Type: application/x-www-form-urlencoded

username=asannameg@gmail.com&password=ParolJok6#
```

### Формат ответа (успех):

```json
{
  "access_token": "eyJ...",
  "token_type": "bearer",
  "user": {
    "id": 1,
    "email": "asannameg@gmail.com",
    "full_name": "asannameg",
    "is_admin": true,
    ...
  }
}
```

### Формат ответа (ошибка):

```json
{
  "detail": "Неверный email или пароль"
}
```

## 🐛 ВОЗМОЖНЫЕ ПРОБЛЕМЫ:

### Проблема 1: API возвращает HTML вместо JSON

**Причина:** Запросы не доходят до FastAPI, попадают в SPA routing

**Решение:** Исправить `.htaccess` (уже исправлено)

### Проблема 2: Пользователь не существует

**Причина:** Пользователь не создан в базе данных

**Решение:** Запустить `create_superadmin.py`

### Проблема 3: Неправильный пароль

**Причина:** Пароль не совпадает с хешем в БД

**Решение:** Пересоздать пользователя через `create_superadmin.py`

### Проблема 4: База данных не подключена

**Причина:** `DATABASE_URL` не настроен или БД недоступна

**Решение:** Проверить переменные окружения и подключение к БД
