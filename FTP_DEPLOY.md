# Инструкция по загрузке на Spaceship через FTP

## FTP подключение

**Данные для подключения:**
- **Host:** ftp.spaceship.ru (или IP адрес сервера Spaceship)
- **Username:** www.zakup.one
- **Password:** ParolJok9@
- **Port:** 21

## Используйте FTP клиент

Рекомендуемые программы:
- **FileZilla** (бесплатно, кроссплатформенно)
- **Cyberduck** (Mac/Windows)
- **WinSCP** (Windows)

## Структура файлов на сервере

После загрузки структура должна быть такой:

```
/ (корень сайта)
├── .htaccess
├── wsgi.py
├── .env (создайте на основе env.production.example)
├── requirements.txt
├── alembic.ini
├── app/
│   ├── __init__.py
│   ├── main.py
│   ├── core/
│   ├── api/
│   ├── models/
│   └── services/
├── alembic/
│   └── versions/
├── frontend/
│   └── dist/
│       ├── index.html
│       └── assets/
├── uploads/ (создайте папку)
└── zakup.db (если используете SQLite)
```

## Пошаговая инструкция

### 1. Подготовка локально

```bash
# Соберите frontend
cd frontend
npm install
npm run build
cd ..

# Запустите скрипт подготовки
./deploy.sh
```

### 2. Подключение через FTP

1. Откройте FTP клиент
2. Создайте новое подключение:
   - Host: ftp.spaceship.ru
   - Username: www.zakup.one
   - Password: ParolJok9@
   - Port: 21
3. Подключитесь

### 3. Загрузка файлов

Загрузите все содержимое папки `deploy/` в корень сайта на сервере:

- **Все файлы из deploy/** → **/** (корень)
- **app/** → **app/**
- **frontend/dist/** → **frontend/dist/**
- **alembic/** → **alembic/**

### 4. Создание необходимых папок

На сервере создайте папки:
- `uploads/` - для загрузки прайс-листов
- `downloads/` - если используется

### 5. Настройка прав доступа

Установите права:
- **Папки:** 755
- **Файлы:** 644
- **uploads/:** 777 (для записи файлов)
- **zakup.db:** 666 (если используется SQLite)

### 6. Создание .env файла

1. Скопируйте `env.production.example` в `.env`
2. Отредактируйте `.env`:
   - Измените `SECRET_KEY` на случайную строку (минимум 32 символа)
   - Настройте `DATABASE_URL` если используете PostgreSQL
   - Проверьте `CORS_ORIGINS` для вашего домена

### 7. Настройка в панели Spaceship

1. Зайдите в панель управления Spaceship
2. Найдите настройки Python приложения
3. Укажите:
   - **Точка входа (Entry Point):** `wsgi:application`
   - **Версия Python:** 3.11 (или доступная на сервере)
4. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
   (или через панель управления, если есть такая возможность)

### 8. Настройка домена

В панели Spaceship:
1. Добавьте домен `www.zakup.one` или `zakup.one`
2. Настройте DNS записи (если нужно):
   - A запись: IP адрес сервера Spaceship
   - CNAME для www: основной домен

### 9. Проверка

После загрузки проверьте:
1. Главная страница: `https://www.zakup.one`
2. API health: `https://www.zakup.one/api/v1/health`
3. API docs: `https://www.zakup.one/api/docs`

## Важные замечания

⚠️ **SECRET_KEY**: ОБЯЗАТЕЛЬНО измените на случайную строку в `.env`!

⚠️ **База данных**: Если используете SQLite, убедитесь что файл `zakup.db` доступен для записи

⚠️ **HTTPS**: После настройки SSL сертификата, раскомментируйте правила HTTPS в `.htaccess`

⚠️ **CORS**: Убедитесь что `CORS_ORIGINS` в `.env` содержит ваш домен

## Обновление сайта

При обновлении:
1. Соберите frontend: `cd frontend && npm run build`
2. Запустите `./deploy.sh`
3. Загрузите обновленные файлы через FTP
4. Перезапустите приложение в панели Spaceship (если нужно)

## Решение проблем

### Ошибка 500
- Проверьте логи в панели Spaceship
- Убедитесь что все зависимости установлены
- Проверьте права доступа к файлам

### API не работает
- Проверьте `.htaccess` файл
- Убедитесь что `wsgi.py` настроен правильно
- Проверьте что Python приложение запущено

### Статические файлы не загружаются
- Проверьте путь к `frontend/dist/`
- Убедитесь что файлы загружены
- Проверьте права доступа
